## 한국어

### 1. 개요
PlantCare(이하 "앱")는 사용자가 기르는 반려식물을 기록하고 물주기 알림을 받을 수 있도록 돕는 모바일 애플리케이션입니다. 본 방침은 앱이 어떤 정보를 처리하는지, 왜 처리하는지, 사용자가 어떤 권리를 행사할 수 있는지를 설명합니다.

### 2. 수집·처리하는 정보
앱은 계정을 요구하지 않으며 서버로 개인정보를 전송하지 않습니다. 다음 정보는 **사용자 기기 내부에서만** 저장·처리됩니다.

| 구분 | 항목 | 저장 위치 |
|---|---|---|
| 식물 정보 | 이름, 종, 별명, 장소, 획득일, 물주기/비료 주기, 마지막 물준 날짜 등 | 기기 내부 Room DB |
| 케어 기록 | 물주기/비료/분갈이/가지치기/사진/메모 로그 | 기기 내부 Room DB |
| 사진 | 식물 커버 사진, 성장/케어 로그 사진 | 앱 전용 내부 저장소 |
| 환경 설정 | 알림 시각, 테마(라이트/다크/시스템) | DataStore Preferences |

앱은 위 정보를 외부 서버로 전송하거나 수집하지 않습니다.

### 3. 권한 사용 목적
- **카메라 (CAMERA)**: 식물 사진을 직접 촬영해 커버/케어 로그에 첨부하기 위해 사용합니다. 촬영한 이미지는 기기 내부에만 저장됩니다.
- **사진/미디어 접근 (READ_MEDIA_IMAGES, Android 13+ PHOTO_PICKER)**: 사용자가 선택한 갤러리 사진을 앱 내부 저장소로 복사하기 위해 사용합니다. 사용자가 선택하지 않은 파일에는 접근하지 않습니다.
- **알림 (POST_NOTIFICATIONS)**: 설정한 시각에 식물별 물주기 알림을 표시하기 위해 사용합니다.
- **인터넷 (INTERNET)**: 현재 버전에서는 광고 SDK가 포함되지 않아 네트워크를 사용하지 않습니다. 추후 광고(예: Google AdMob)가 포함되면 해당 시점의 본 방침 버전에서 명시합니다.

### 4. 제3자 제공 및 외부 SDK
- 본 앱은 현재 버전에서 사용자 데이터를 제3자에게 제공하지 않습니다.
- **광고 (추후 추가 예정)**: Google AdMob이 적용되면 광고 ID 등 제한된 식별자가 Google에 전송되어 광고 제공·최적화에 활용될 수 있습니다. 해당 기능이 추가되는 버전부터 본 방침에 상세 내용과 옵트아웃 방법을 반영합니다. Google의 광고 처리에 대해서는 [Google 광고 정책](https://policies.google.com/technologies/ads)을 참조하세요.

### 5. 데이터 보관 및 삭제
- 모든 데이터는 기기 내부에만 존재하며, 사용자가 앱을 삭제하면 함께 제거됩니다.
- 사용자는 앱 내 설정에서 JSON 파일로 데이터를 내보내거나(백업), 동일 형식의 파일로 가져올 수 있습니다(복원). 내보낸 파일은 사용자가 선택한 위치에 저장되며, 이후 관리 책임은 사용자에게 있습니다.
- 앱 내에서 식물 또는 케어 로그를 삭제하면 해당 기록은 복구할 수 없습니다.

### 6. 아동의 개인정보
본 앱은 13세 미만 아동을 대상으로 설계되지 않았습니다. 아동으로부터 고의로 개인정보를 수집하지 않습니다.

### 7. 보안
앱은 사용자 데이터를 기기 내부 저장소에만 보관합니다. 외부 서버 전송이 없으므로 서버 측 보안 사고 위험은 존재하지 않으나, 기기 자체의 보안(잠금화면, 백업 암호화 등)은 사용자의 관리에 따릅니다.

### 8. 방침 변경
본 방침은 기능 추가(특히 광고 SDK 추가 등) 시 개정될 수 있습니다. 중요한 변경이 있는 경우 앱 내 또는 본 문서의 "최종 수정일"을 통해 고지합니다.

## English

### 1. Overview
PlantCare ("the App") is a mobile application that helps users record their houseplants and receive watering reminders. This policy explains what information the App processes, why, and the rights users may exercise.

### 2. Information the App Processes
The App does not require an account and does not transmit personal information to any server. The following data is stored and processed **only on the user's device**.

| Category | Items | Storage |
|---|---|---|
| Plant data | Name, species, nickname, location, acquired date, watering/fertilizer intervals, last watered/fertilized dates | On-device Room database |
| Care logs | Water / fertilizer / repot / prune / photo / note entries | On-device Room database |
| Photos | Cover photos, care-log photos | App-private internal storage |
| Preferences | Notification time, theme (light/dark/system) | DataStore Preferences |

None of the above is collected or sent to any external server by the App.

### 3. Permissions
- **CAMERA**: To capture plant photos for cover images and care logs. Captured images are saved only on the device.
- **READ_MEDIA_IMAGES (Android 13+ Photo Picker)**: To copy user-selected gallery photos into the app's internal storage. The App does not access images the user has not selected.
- **POST_NOTIFICATIONS**: To display per-plant watering reminders at the scheduled time.
- **INTERNET**: The current version does not include an advertising SDK and does not use the network. If advertising (e.g., Google AdMob) is added in a future version, this policy will be updated accordingly.

### 4. Third Parties and External SDKs
- The current version does not share user data with third parties.
- **Advertising (planned)**: When Google AdMob is added, a limited set of identifiers (such as the advertising ID) may be transmitted to Google for ad delivery and measurement. The policy will be updated at that time with details and opt-out instructions. See [Google Ads policies](https://policies.google.com/technologies/ads).

### 5. Data Retention and Deletion
- All data resides only on the device and is removed when the App is uninstalled.
- Users can export data as a JSON file (backup) and re-import it in the same format (restore). The exported file is stored in the user-selected location; its subsequent handling is the user's responsibility.
- Deleting a plant or care log in the App is permanent and cannot be undone.

### 6. Children's Privacy
The App is not designed for children under 13 and does not knowingly collect personal information from children.

### 7. Security
Data is kept only in the device's internal storage. Because no data is sent to a server, server-side breach risk does not apply; however, device-level security (screen lock, backup encryption, etc.) is the user's responsibility.

### 8. Changes to This Policy
This policy may be updated when features change — in particular, when an advertising SDK is added. Material changes will be indicated by updating the "Last updated" date at the top of this document, or in-app where appropriate.
